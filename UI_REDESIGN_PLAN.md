# 星渊笔 · 前端 UI 重设计计划书

> 版本 v1.0 · 基于 `xingyuanbi` 仓库快照（2026-07-11）
> 定位：个人本地 AI 长篇小说创作系统 · 单作者专注写作工具

---

## 0. 计划书边界

- **本文件仅为设计计划书**，不修改仓库任何现有代码。
- 计划基于仓库实际构成：Dashboard / Workspace（三栏：ChapterRail + EditorPanel + ContextPanel）/ Bible（人物 + 世界设定）/ Review / Outline / Settings / NewBookWizard / AgentChatPanel / AiConsole（generate/quality/voice/ironlock/foreshadow/history 多 tab）/ GlobalSearch / 历史快照 / 主题系统（light/dark/abyss）。
- 技术栈不变：Vue 3 + TS + Vite + Pinia + Naive UI + Tiptap。本计划是**视觉与交互层**的重设计，不替换框架。

## 1. 设计总纲

### 1.1 视觉论题（Visual Thesis）

> **「墨渊手稿」** —— 在一座午夜的私人观象台里写作。
> 墨色星渊为底，宋体为骨，星光青为唯一锐点。界面如同一卷摊开的线装手稿，安静、深邃、可长坐。

一句话：**Editorial restraint（编辑级克制）× Literary serif（文学衬线）× Cosmic depth（星渊深度）**。

### 1.2 为什么是这个方向

| 维度 | 现状问题 | 重设计回应 |
|---|---|---|
| 品牌色 | `#7c6cbf` 紫色 + Naive UI 默认 = 通用 AI slop | 弃紫，改「星渊青」`#6FE3D4`（暗）/ `#0E7C7B`（亮），唯一锐点 |
| 字体 | 系统无衬线，与"文学创作"语境割裂 | 思源宋体（display）+ 思源黑体（UI）+ JetBrains Mono（元数据） |
| 布局 | 卡片堆叠 + Naive UI 默认 chrome | Linear 式克制：去卡片、用分隔线与留白、Editor 为视觉锚点 |
| 文字色 | 纯白 `#FFF` 刺眼，长写疲劳 | 暖象牙 `#E8E4D8`（暗）/ 米白 `#FBF8F0`（亮） |
| 氛围 | 纯色背景，无深度 | 极淡星点纹理 + 墨色渐变（仅暗主题），不喧宾夺主 |
| 动效 | 零或零散 | 3 处刻意动效：入场墨显、AI 流式墨入、质量环呼吸 |

### 1.3 设计原则（不可妥协）

1. **Editor 是英雄**：写作区是第一视觉锚点，其余皆为伺服。任何元素若削弱编辑器焦点，删之。
2. **去卡片化**：卡片仅当「卡片本身是交互对象」时使用（人物卡、故事线节点）。其余一律改用分隔线、留白、字阶层级。
3. **单锐点配色**：一个主色（星渊青）贯穿，danger/warning 仅在状态指示时出现，不做装饰。
4. **两种字体上限**：宋体（display）+ 黑体（UI），加一种 mono 用于元数据。无第四种。
5. **Utility copy 优先**：产品界面用操作语言（"当前章节 · 第 12 章"），不用营销语言（"开启你的创作之旅"）。
6. **可长坐**：长文写作场景，对比度、字号、行距、留白均按"连续 4 小时阅读不疲劳"校准。

### 1.4 一句话记忆点

> 别人记不住第二个用紫色渐变的写作工具，但会记住**「在墨色星渊里用宋体写作」**的那一个。

## 2. 视觉系统

### 2.1 色彩系统

弃用现有 `--xy-*` 紫色 token 体系，重建为「墨渊」色板。三套主题共享语义 token，仅数值不同。

#### 2.1.1 Abyss（暗主题 · 默认）

```css
/* 墨色星渊 - 暗主题默认 */
--ink-void:      #0B0E14;  /* 背景-基底：墨黑带蓝调，非纯黑 */
--ink-surface:   #141821;  /* 背景-表面：面板/侧栏 */
--ink-elevated:  #1C2230;  /* 背景-悬浮：抽屉/弹层 */
--ink-line:      #2A3040;  /* 分隔线，极淡 */
--paper-primary: #E8E4D8;  /* 正文：暖象牙，非纯白，长写不刺眼 */
--paper-muted:   #8A8678;  /* 次要文字：灰褐 */
--paper-faint:   #4A4D56;  /* 占位/禁用 */
--starlight:     #6FE3D4;  /* ★ 唯一锐点：星渊青 */
--starlight-dim: #2A4A47;  /* 锐点弱化态/底色 */
--ember:         #E3A56F;  /* 警示（伏笔待收、文风漂移）*/
--crimson:       #E37F6F;  /* 危险（删除、红线违规）*/
--jade:          #7FB283;  /* 通过（审查 pass、章节完成）*/
```

#### 2.1.2 Paper（亮主题）

```css
--paper-void:    #F5F1E8;  /* 背景：陈年米纸，非纯白 */
--paper-surface: #FBF8F0;  /* 表面 */
--paper-elevated:#FFFFFF;  /* 悬浮 */
--paper-line:    #E0DACA;  /* 分隔线 */
--ink-primary:   #1A1A1A;  /* 正文 */
--ink-muted:     #5C5852;  /* 次要 */
--ink-faint:     #A8A29A;  /* 占位 */
--starlight:     #0E7C7B;  /* 锐点：加深青，保证亮底对比度 */
--starlight-dim: #C9E4E2;
```

#### 2.1.3 Dusk（第三主题 · 替换原 abyss）

将原 abyss 主题重新定义为「暮色」——暖色暗主题，色温偏琥珀，给作者另一种情绪选择。

```css
--dusk-void:     #1A1410;  /* 暖墨褐 */
--dusk-surface:  #241C16;
--dusk-elevated: #2E251D;
--dusk-line:     #3A2E24;
--paper-primary: #EDE0D0;  /* 暖象牙偏黄 */
--paper-muted:   #9A8B78;
--starlight:     #E3A56F;  /* 琥珀星光，与暗主题青形成主题差异 */
```

> **三主题记忆点**：Abyss=冷墨青星 / Paper=陈纸墨字 / Dusk=暖暮琥珀。同一套语义 token，三种情绪。

### 2.2 字体系统

| 角色 | 字体 | 用途 | 备注 |
|---|---|---|---|
| **Display** | Noto Serif SC（思源宋体）+ Cormorant Garamond（拉丁数字/章节号） | 品牌名、章节标题、H1-H3、空状态大字 | 文学语境，高对比笔画 |
| **UI Body** | Noto Sans SC（思源黑体） | 正文 UI、按钮、表单、对话 | 中性可读 |
| **Mono** | JetBrains Mono | 字数、章节号、质量分、时间戳、token 计数 | 等宽，技术感元数据 |

**字阶（Type Scale）**：

```
Display XL  56px / 1.1  · 宋体 · 品牌名、空状态
Display L   40px / 1.15 · 宋体 · 章节标题
Display M   28px / 1.2  · 宋体 · 区块大标题
Heading     20px / 1.3  · 黑体 · 区块标题
Body L      17px / 1.75 · 宋体 · ★ 编辑器正文（长文阅读最优）
Body        15px / 1.6  · 黑体 · UI 正文
Body S      13px / 1.5  · 黑体 · 次要 UI
Mono        13px / 1.4  · Mono · 元数据
Mono S      11px / 1.3  · Mono · 标签（uppercase + letter-spacing 0.08em）
```

**关键决策**：编辑器正文用 **17px 宋体 / 行高 1.75**——这是中文长文阅读的黄金组合，比当前无衬线更贴合"小说"语境。

### 2.3 间距与栅格

- 基础单位 `4px`，采用 `4 / 8 / 12 / 16 / 24 / 32 / 48 / 64` 八档。
- 编辑器正文列宽 **`max-width: 72ch`** 居中——印刷品最佳行宽。
- 工作台三栏：ChapterRail `220px`（现 240，收窄）/ Editor `1fr` / ContextPanel `360px`（现 320，加宽以承载质量环+记忆锁）。
- 全局左右安全边距：桌面 `48px`，移动 `20px`。

### 2.4 质感与氛围（仅暗主题）

克制使用，避免噪点干扰写作：

1. **星点纹理**：`background-image` 用极淡径向渐变模拟星点，`opacity: 0.03`，仅 body 层。
2. **墨色渐变**：body 顶到底 `ink-void → #090A10`，制造纵深。
3. **编辑器纸感**：编辑区背景比 surface 亮一档（`#161A24`），两侧留 1px 极淡 line，如手稿纸页。
4. **禁用**：玻璃拟态大范围模糊、彩色渐变、霓虹光晕——均与"克制"冲突。

### 2.5 图标系统

- 沿用 `@vicons/tabler`，但**统一描边 1.5px**，尺寸 18/20/24 三档。
- 状态点用**实心圆 + 描边**双态：completed = 实心 jade / planned = 描边 starlight / draft = 实心 paper-faint。
- 质量分环：SVG `stroke-dasharray` 动画，色随分段（<60 crimson / 60-79 ember / ≥80 starlight）。

## 3. 信息架构与页面计划

### 3.1 全局壳（AppLayout）

**现状**：左侧深空玻璃拟态侧栏 + 顶部栏 + 内容区。
**重设计**：

```
┌─────────────────────────────────────────────────┐
│ ◆ 星渊笔            第 12 章 · 3,847 字 · ●已存 │ ← 顶栏 48px，极简
├──────┬──────────────────────────────────────────┤
│ 📚   │                                          │
│ ✦写作│                                          │
│ 👥人物│           主内容区                       │
│ 📖圣经│                                          │
│ ⚖审查│                                          │
│ ⚙设置│                                          │
└──────┴──────────────────────────────────────────┘
  64px              1fr
```

- **左导航收窄至 64px**（图标 + 工具提示），腾出写作空间。
- **顶栏 48px**：左侧品牌标（宋体「星渊笔」+ ◆ 符号），右侧 mono 元数据（当前章节/字数/保存状态），最右主题切换。
- **去玻璃拟态**：侧栏纯 `ink-surface`，右侧 1px line 分隔。
- **品牌名是顶栏最响的字**——宋体 20px，其余元数据 mono 13px muted。

### 3.2 Dashboard（首页）

**现状**：4 模块卡片切换（新建/我的小说/审查/世界观）。
**重设计**：编辑式封面网格，非卡片拼贴。

- **Hero 区**：满屏首视口。左侧大字宋体「星渊笔」+ 副标「在墨色星渊里写一部长篇」，右侧一张氛围图（书籍/星空意象，非装饰纹理）。
- **我的小说**：以**书脊式纵向列表**呈现，非卡片网格。每行：书名（宋体 20px）+ 章节进度（mono）+ 质量环（mini）+ 最后写作时间。hover 时整行左移 8px + starlight 左边框亮起。
- **快捷入口**（新建/审查/世界观）降级为顶部三个文字链接，非巨型卡片。
- **空状态**：宋体大字「尚未落笔」+ 一句引导 + 单按钮「新建第一部」。

### 3.3 Workspace（工作台 · 核心）

三栏，但每栏重新定义职责与视觉权重。

#### 3.3.1 ChapterRail（左栏 · 220px）

**现状**：搜索 + 分卷折叠列表 + 状态点。
**重设计**：纵向时间轴隐喻。

```
第 一 卷 · 潜龙在渊
│
●  01  深渊之始              3,847
│
●  02  云屿之盟              4,102
│
○  03  渊海之下              —     ← planned，描边圆
│
•  04  （草稿）              1,200  ← draft，实心灰
```

- 章节号用 **mono + 灰**，标题用宋体，字数用 mono muted 右对齐。
- 当前章节整行 `starlight-dim` 底色 + 左 2px starlight 竖线。
- 搜索框置顶，无图标时仅占位「⌘K 检索章节」。

#### 3.3.2 EditorPanel（中栏 · 英雄）

**现状**：Tiptap + 工具栏 + 行号。
**重设计**：手稿纸感 + 打字机模式。

- **正文区 72ch 居中**，背景比侧栏亮一档（纸感），左右 1px line。
- **正文 17px 宋体 / 行高 1.75 / 段间距 1em**。
- **工具栏默认隐藏**，hover 顶部 32px 区域渐显（focus 模式）。含：撤销/重做/粗斜体/H1-3/对齐/AI 续写。
- **行号**：仅 hover 段落时显示该段行号（mono faint），常驻行号干扰阅读。
- **打字机模式**：光标始终垂直居中（可切，默认开）。
- **AI 续写光标**：流式 token 用 `starlight` 色 + 0.3s 渐入，如墨滴晕开。续写中编辑器底部显示 mono「AI 续写中 · 412 tokens」。
- **字数统计**：不占顶栏，浮动于右下角 mono 小字「3,847 / 目标 2,000」，达标变 jade。

#### 3.3.3 ContextPanel（右栏 · 360px）

**现状**：多 tab 上下文面板。
**重设计**：以质量环为锚的纵向叙事简报。

- **顶部质量环**（视觉锚）：SVG 环 80px，中心 mono 大字分数，下方宋体等级（S/A/B/C）。环色随分段。
- **叙事简报**：宋体小标题 + 黑体正文，分段：故事线进度 / 当前章纲 / 上一章摘要。
- **记忆锁指示**（Iron Lock）：mono 标签「铁锁 · 12 条事实 / 8 节拍 / 3 待收」，hover 展开列表。
- **伏笔快览**：待收伏笔 3 条，每条 ember 圆点 + 标题 + 埋设章节号。
- tab 切换降级为顶部三个文字标签（简报/记忆/伏笔），非按钮组。

### 3.4 AgentChatPanel（AI 对话）

**现状**：抽屉式 + SSE 流式 + tool_call 卡片。
**重设计**：对话如手稿边注（marginalia）。

- **抽屉从右侧滑入**，宽 420px，覆盖 ContextPanel。
- **用户消息**：右对齐，`starlight-dim` 底，宋体 15px。
- **AI 消息**：左对齐，无底色，宋体 15px，流式 token 0.2s 渐入。
- **tool_call 卡片**：极简化——mono 小字「调用 generate_chapter」+ 折叠的参数 JSON（默认收起），结果用 jade/ember 状态点。不用大卡片。
- **空态**：宋体大字「与星渊对话」+ 三个建议项（生成第 3 章 / 审查当前章 / 查询人物），文字链接式，非按钮。
- **停止生成**：底部居中 crimson 圆点 + 「停止」mono 小字。

### 3.5 AiConsole（AI 控制台）

**现状**：generate/quality/voice/ironlock/foreshadow/history 多 tab。
**重设计**：从「多 tab 设置面板」转为「命令台」。

- **触发**：顶栏「✦ 控制台」按钮，全屏遮罩 + 居中浮层（非侧栏 tab）。
- **布局**：左侧 180px 文字命令列表（生成/质检/文风/铁锁/伏笔/历史），右侧参数面板。
- **参数控件**：滑块用极简横线 + starlight 圆点（非 Naive 默认粗滑块）；数值 mono 右对齐。
- **自动驾驶模式**：顶部醒目 toggle，开启时顶栏出现「自动驾驶中」脉动 starlight 指示。
- **执行**：底部「执行」按钮，starlight 实心，宋体 15px。

### 3.6 Bible（设定圣经）

**现状**：左 tab（人物/设定）+ 右详情。
**重设计**：人物卡 = 塔罗式肖像卡（卡片在此是交互对象，合规）。

- **人物卡**：纵向卡片，顶部色块（角色主题色）+ 宋体大名 + 黑体身份标签 + 五维雷达图（mini，60px）。hover 卡片轻微上浮 + starlight 描边。
- **人物详情**：右侧抽屉，宋体大字姓名 + 分段（外貌/性格/背景/关系），关系用极简连线图（节点 = 关联角色）。
- **世界设定**：树形左栏 + 右栏 Markdown 渲染，去卡片，用分隔线分节。

### 3.7 Review（审查页）

**现状**：章节审查页。
**重设计**：审查报告式排版。

- **顶部**：章节标题（宋体）+ 总分大字（mono 40px）+ 等级徽章。
- **维度分**：8 个护栏横向 mono 小条形图，每条带分数 + 状态色。
- **红线违规**：若有，crimson 区块逐条列出，含位置（段落/字偏移）+ 建议。
- **详情**：宋体正文段落，问题处 ember 下划线，hover 浮层显示建议。

### 3.8 GlobalSearch（⌘K）

**现状**：模态 + 5 类结果。
**重设计**：Spotlight 式。

- 居中浮层，`ink-elevated` 底，starlight 1px 边。
- 输入框无背景，仅底部 1px starlight 线。
- 结果分组：人物 / 设定 / 伏笔 / 事实 / 章节，组标题 mono uppercase 小字。
- 每条结果：宋体标题 + mono 路径（如「人物 · 林渊」）+ 匹配片段高亮（starlight-dim 底）。

### 3.9 NewBookWizard（新建向导）

**现状**：向导式。
**重设计**：三步极简表单，每步满屏居中。

- 步骤指示：顶部 mono「01 / 03」+ 宋体步骤名。
- 输入框：无背景，仅底部 1px line，focus 时 starlight。
- 第二步可选「导入设定草稿」用文字链接，非按钮。
- 完成页：宋体大字「《书名》已就绪」+ 单按钮「开始第一章」。

### 3.10 Settings（设置）

**现状**：设置页。
**重设计**：分节列表，非卡片。

- 左侧分节导航（AI 配置 / 生成参数 / 外观 / 数据 / 关于），右侧表单。
- **AI 配置**：Provider 选择用文字单选（OpenAI / Anthropic / Ollama / 本地），API Key 输入框带「测试连接」文字链接。
- **外观**：三主题预览（Abyss / Paper / Dusk）用三个小色块圆角矩形，选中 starlight 描边。
- 所有控件去 Naive 默认粗样式，改极简线框。

## 5. 交互与动效计划

按 frontend-skill 要求，**3 处刻意动效**，余皆静止。

### 5.1 入场墨显（Page Load）

工作台首次加载：编辑器正文按段落 `staggered reveal`，每段 `translateY(8px) + opacity 0→1`，间隔 60ms，缓动 `cubic-bezier(0.22, 1, 0.36, 1)`，总时长 < 1.2s。仅首次，路由切换不重复。

### 5.2 AI 流式墨入（Streaming）

- 编辑器续写：每个 token `color: starlight` → 0.3s 渐变至 `paper-primary`，如墨滴晕开。
- Agent 对话：token 0.2s `opacity 0→1`，无位移，安静。
- 流式中：光标处 1px starlight 竖线 1s 呼吸脉动。

### 5.3 质量环呼吸（Quality Ring）

审查完成时，质量环 `stroke-dashoffset` 从满到目标值 1.2s 动画，缓动 `ease-out`，完成后中心分数 mono `opacity 0→1`。分数变化时环色平滑过渡。

### 5.4 其余微交互（克制）

- hover：卡片/列表项 `translateX(4px)` 或 `translateY(-2px)`，200ms，不缩放。
- 按钮按下：`active` 时 `translateY(1px)`，无阴影变化。
- 路由切换：`fade 150ms`，无滑动。
- **禁用**：视差滚动、3D 翻转、弹跳、spring 过冲——与"长坐专注"冲突。

## 9. 验收标准（Litmus Checks）

- [ ] 编辑器是第一视觉锚点——隐藏其余 UI，首屏仍能认出"这是写作工具"。
- [ ] 品牌名「星渊笔」在顶栏最响——隐藏导航后仍知道在哪个产品。
- [ ] 仅扫标题/元数据即可理解当前页面状态（章节号、字数、质量分、保存状态）。
- [ ] 全站仅一个锐点色（星渊青），无第二个主色竞争。
- [ ] 全站仅两种字体（宋体 + 黑体）+ 一种 mono，无第四种。
- [ ] 卡片仅出现在人物卡、故事线节点等"卡片即交互"处，其余无卡片。
- [ ] 移除所有装饰阴影后，设计仍显克制高级。
- [ ] 连续写作 4 小时无视觉疲劳（正文 17px 宋体 / 1.75 行高 / 暖象牙字色）。
- [ ] 三主题切换无色值硬编码残留（全走语义 token）。
- [ ] 动效仅 3 处刻意，余皆静止，无视觉噪声。

## 10. 风险与取舍

| 风险 | 取舍 |
|---|---|
| 宋体渲染性能（长文档） | 编辑器正文宋体，UI 仍黑体；超长章节（>5万字）提供"性能模式"切回黑体 |
| Naive UI 默认样式覆盖成本高 | 优先用 `theme-overrides` + CSS 变量，不改组件源码；顽固组件自封装替代 |
| 字体自托管体积 | Noto Serif/Sans SC 按字重子集化，仅加载 Regular + Medium，woff2 约 2-4MB |
| 主题 token 迁移破坏现有 | 阶段 1 先建 token 双轨（旧 `--xy-*` 与新语义并存），组件逐步迁移，最后移除旧 token |
| 移动端打字机模式不可用 | 移动端默认关闭，不报错 |
