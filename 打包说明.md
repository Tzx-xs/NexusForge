# NexusForge Windows 桌面应用打包说明

## 方案一：GitHub Actions 自动打包（推荐）

无需本地安装任何开发环境，推送到 GitHub 即可自动打包。

### 步骤

1. **推送项目到 GitHub**
   ```bash
   cd NexusForge
   git init
   git add .
   git commit -m "NexusForge 融合完成"
   git remote add origin https://github.com/你的用户名/NexusForge.git
   git push -u origin main
   ```

2. **手动触发打包**
   - 打开 GitHub 仓库页面
   - 点击 `Actions` 标签
   - 左侧选择 `Build Windows Desktop App`
   - 点击 `Run workflow` → 选择 `main` 分支 → 点击绿色按钮

3. **下载产物**
   - 等待打包完成（约 15-30 分钟）
   - 点击完成的 workflow run
   - 在页面底部 `Artifacts` 区域下载 `NexusForge-windows-installer`（NSIS 安装包）或 `NexusForge-windows-exe`（便携版）

4. **（可选）打 tag 自动发布 Release**
   ```bash
   git tag v1.0.0
   git push origin v1.0.0
   ```
   GitHub 会自动创建 Release 并附带 .exe 安装包

## 方案二：Windows 本地打包

需要本地安装开发环境，但打包速度快（首次约 20 分钟，增量约 5 分钟）。

### 前置依赖安装

| 依赖 | 下载地址 | 安装要点 |
|------|---------|---------|
| Python 3.14 | https://www.python.org/downloads/ | 勾选 `Add to PATH` |
| Node.js 20+ | https://nodejs.org/ | LTS 版本即可 |
| Rust | https://rustup.rs/ | 默认安装 |
| MSVC Build Tools | Visual Studio Installer | 勾选「使用 C++ 的桌面开发」 |
| WebView2 Runtime | https://developer.microsoft.com/microsoft-edge/webview2/ | Win11 自带，Win10 需安装 |

### 执行打包

```powershell
# 在项目根目录执行
.\scripts\build_windows.ps1
```

脚本会自动完成：
1. 安装后端 Python 依赖 + PyInstaller
2. 安装前端 npm 依赖
3. 构建前端（Vue 3 + Tiptap）
4. PyInstaller 打包后端为单文件
5. Tauri 打包桌面应用（NSIS 安装包）

### 产物位置

```
frontend\src-tauri\target\release\bundle\nsis\NexusForge_1.0.0_x64-setup.exe  # NSIS 安装包
frontend\src-tauri\target\release\nexusforge.exe                              # 便携版（需配合 nexusforge-backend 目录）
```

## 安装与运行

### NSIS 安装包（推荐）
1. 双击 `NexusForge_1.0.0_x64-setup.exe`
2. 按提示选择安装目录（默认 `C:\Program Files\NexusForge`）
3. 安装完成后从开始菜单启动 `NexusForge`

### 便携版
1. 将 `nexusforge.exe` 和 `out/tauri/nexusforge-backend/` 目录放在同一父目录下
2. 双击 `nexusforge.exe` 运行

### 首次运行
- 应用会自动启动内置 Python 后端（无需手动配置）
- 数据存储在 `%APPDATA%\com.nexusforge.desktop\data\`
- AI 写作功能需在「设置 → AI 参数」中配置 LLM API Key

## 常见问题

### Q: 打包失败提示 "linker 'link.exe' not found"
A: 未安装 MSVC Build Tools。打开 Visual Studio Installer，勾选「使用 C++ 的桌面开发」工作负载。

### Q: 运行时白屏
A: 未安装 WebView2 Runtime。从 https://developer.microsoft.com/microsoft-edge/webview2/ 下载安装。

### Q: 后端启动失败
A: 检查杀毒软件是否拦截了 `nexusforge-backend.exe`，将其加入白名单。

### Q: GitHub Actions 打包失败
A: 查看 Actions 日志，常见原因是依赖版本冲突。可重新触发 workflow。
